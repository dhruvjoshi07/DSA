    2. Exceptions Ke Do Main Types (The Big Family)Java me exceptions do bade 
        groups me divided hote hain:
    🟩 Checked Exceptions (Compile-time)Yeh wo galtiyan hain jo Java ka compiler code likhte waqt hi pakad leta hai. Wo bolta hai:
         "Pehle isko handle karo, warna code chalne hi nahi dunga." (Yeh mostly tab aate hain jab hum computer se bahar ki duniya se 
         interact karte hain, jaise files ya internet).
    *IOException:
         Jab file read ya write karte waqt koi dikkat aaye (jaise file delete
          ho gayi ho).
    *FileNotFoundException:
         Jab tum kisi aisi file ko open karne ki koshish karo jo computer me exist hi nahi karti.
    🟥 Unchecked Exceptions (Runtime)Yeh wo galtiyan hain jo compile toh aaram se ho jati hain, par jab program actual me chal raha
         hota hai tab sudden blast karti hain. (Yeh mostly developer ki logic ki galti hoti hain).
    *NullPointerException (NPE): Java ka 
         sabse bada villain! Jab tum kisi aisi cheez (object) par kaam karne ki koshish karo jo null hai (yani memory me bani hi 
         nahi).
        
    *ArrayIndexOutOfBoundsException: Jab tum array ke size se bahar ka element access karo. Jaise size 5 hai aur tum array[10]
          mang rahe ho.ArithmeticException: Jab tum math ka koi illegal rule todte ho, jaise kisi number ka divide by zero ($5 / 0$) karna.

Factorial me banaya). Agar kisi ne rule toda, toh tum khud bolte ho throw new Exception().Javapublic void setAge(int age) {

💡 Pro-Tip for your Code:Tumne apne main method ke aakhiri catch block me catch (Exception e) lagaya hai.
     Yeh ek "Catch-All" net ki tarah hai. Agar InputMismatchException ke alawa koi aur naya gaddha 
     (jaise out of memory) aaya, toh yeh Exception e use bhi pakad lega aur program ko crash hone se bacha lega!
